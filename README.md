# M5 AI Dictation Remote

M5StickC Plus2 превращается в беспроводной пульт голосового ввода. Звук передаётся на компьютер с
Windows, локально распознаётся с помощью faster-whisper, копируется или вставляется в активное
приложение и при необходимости отправляется настраиваемой комбинацией клавиш. OpenAI API и облачные
сервисы распознавания речи не используются.

## Готовая версия

Для обычного использования скачайте из GitHub Releases архив
`M5AIDictationServer-<версия>-windows.zip`, распакуйте его и запустите
`M5AIDictationServer.exe`. Архив уже содержит пакет прошивки для USB-обновления M5StickC Plus2;
модель распознавания загружается приложением отдельно при первом выборе.

Подробная установка, настройка, управление, прошивка и устранение неисправностей собраны в
[руководстве пользователя](docs/USER_GUIDE.md).

## Запуск из исходников

1. Установите Python 3.11, 3.12 или 3.14 и PlatformIO Core.
2. Выполните `powershell -ExecutionPolicy Bypass -File .\scripts\setup_windows.ps1`.
3. Проверьте `pc_server\config.yaml`. Полное приложение запускается из исходников командой
   `.\scripts\start_tray.ps1` или задачей VS Code `M5: Запустить приложение из исходников`. Консольный
   сервер без GUI запускается через `.\scripts\start_server.ps1`.
4. Соберите пакет командой `.\scripts\build_firmware.ps1`, запустите главное приложение и нажмите
   «Прошивка…» в карточке устройства. Для консольной работы из исходников остаётся
   `.\scripts\flash_firmware.ps1`.
5. При первом запуске подключитесь к `M5-AI-Remote-XXXX`, откройте `http://192.168.4.1` и укажите
   параметры Wi-Fi и сервера.
6. Установите курсор в любое текстовое поле и говорите, удерживая кнопку M5, либо дважды коротко
   нажмите M5 для автоматической диктовки по паузам.

Структура проекта:

- `firmware` — прошивка PlatformIO для M5StickC Plus2;
- `pc_server` — локальный сервер распознавания и Windows-интерфейс;
- `scripts` — установка, сборка, упаковка релиза и USB-прошивка;
- `tools` — вспомогательные инструменты разработчика.

Теги вида `v*` запускают GitHub Actions: прошивка и Windows-приложение собираются на чистом runner,
после чего ZIP-архивы и `SHA256SUMS.txt` публикуются в GitHub Releases. Исходный код отдельно
скачивается стандартными архивами GitHub или через `git clone`.

Для выпуска сопровождающий проекта должен добавить в GitHub Actions secrets:

- `WINDOWS_SIGNING_CERTIFICATE_BASE64` — PFX-сертификат подписи кода в Base64;
- `WINDOWS_SIGNING_CERTIFICATE_PASSWORD` — пароль PFX.

При необходимости URL RFC 3161-сервера задаётся repository variable
`WINDOWS_SIGNING_TIMESTAMP_URL`; по умолчанию используется сервер DigiCert. Workflow подписывает
EXE до упаковки, проверяет Authenticode-подпись и создаёт GitHub artifact attestations для обоих
ZIP-архивов и файла контрольных сумм. Без сертификата выпуск намеренно завершается ошибкой.

## Лицензия и развитие

Исходный код этого репозитория распространяется по лицензии MIT. Текущая версия бесплатна; уже
опубликованные версии сохраняют условия своей лицензии. В дальнейшем могут появиться отдельные
платные дополнения или сервисы, не входящие в этот репозиторий.

Безопасность: WebSocket-соединение не использует аутентификацию и шифрование. Используйте проект
только в доверенной домашней или частной сети. Разрешайте порт 8765 только для профиля Windows
«Частная сеть» и не открывайте его напрямую в интернет.
