# VoxCortex — руководство пользователя

## Что понадобится

- M5StickC Plus2 и USB-кабель с передачей данных;
- компьютер с 64-битной Windows;
- Wi-Fi 2,4 ГГц, к которому подключены компьютер и устройство;
- доступ в интернет при первой загрузке модели распознавания.

Речь распознаётся локально на компьютере с помощью faster-whisper. OpenAI API и облачные сервисы
распознавания не используются.

## Установка готовой версии

1. Откройте GitHub Releases и скачайте `VoxCortex-<версия>-windows.zip`.
2. Распакуйте архив полностью. Не запускайте EXE прямо из ZIP.
3. Запустите `VoxCortex\VoxCortex.exe`.
4. Разрешите доступ в брандмауэре Windows только для частных сетей.
5. Подключите M5StickC Plus2 по USB, откройте в приложении окно «Прошивка…» и выполните заводскую
   установку для нового устройства либо обычное обновление для уже настроенного.
6. Выберите модель распознавания. При отсутствии локальной копии приложение предложит её скачать.

EXE не подписан платным сертификатом Authenticode, поэтому Windows может показать предупреждение
SmartScreen. Проверить целостность скачанных архивов можно по файлу `SHA256SUMS.txt` и GitHub
artifact attestation из того же Release.

## Первичная настройка устройства

После заводской прошивки подключитесь к сети `VoxCortex-XXXX` и откройте
`http://192.168.4.1`. Укажите:

- имя и пароль домашней сети Wi-Fi 2,4 ГГц;
- IPv4-адрес компьютера или `voxcortex.local`;
- порт `8765`;
- понятное имя устройства.

Удержание B при включении удаляет сетевые настройки и повторно открывает режим настройки.

Точка первоначальной настройки и рабочее WebSocket-соединение пока не используют шифрование и
аутентификацию. Используйте устройство только в доверенной домашней сети, не открывайте порт 8765 в
интернет и не разрешайте его для публичного профиля брандмауэра Windows.

## Управление диктовкой

- Удерживайте M5, говорите и отпустите кнопку для распознавания.
- Дважды коротко нажмите M5, чтобы включить автоматическую диктовку по паузам. Повторное двойное
  нажатие выключает её.
- В автоматическом режиме одиночное нажатие M5 завершает текущую фразу вручную.
- C отменяет текущую запись или операцию. Короткое C в состоянии готовности включает или выключает
  экран; длительное C сохраняет штатное поведение выключения питания.
- Действие выбирается в приложении: `COPY`, `PASTE`, `PASTE + ENTER` или
  `PASTE + CTRL+ENTER`.

Перед диктовкой установите курсор в нужное текстовое поле. Для длинной автоматической диктовки
безопаснее выбирать `PASTE`: варианты с Enter автоматически отправляют каждую законченную фразу.
Автоматическое прослушивание не сохраняется после перезапуска и всегда включается вручную.

## Модели и локальные данные

Доступны модели `Tiny`, `Base`, `Small`, `Medium`, `Large v3` и `Turbo`. Распознавание выполняется на
CPU в режиме int8; CUDA не требуется. Папку моделей можно выбрать через «Настройки → Папка
моделей»; там же модели скачиваются и удаляются.

Пользовательские данные хранятся в `%LOCALAPPDATA%\VoxCortex`:

- `config.yaml` — настройки;
- `history.json` — последние 250 распознанных фраз;
- `logs\server.log` — диагностический журнал;
- `models` — модели по умолчанию; в настройках можно выбрать другой диск или папку;
- `tmp` — временные аудиофайлы.

Эти данные не входят в Git и официальные релизные архивы. При первом запуске прежние настройки,
история и журналы из `%LOCALAPPDATA%\M5AIDictationRemote`, рядом с EXE или в `pc_server` копируются
сюда. Уже скачанные модели не копируются
из-за их размера: приложение сохраняет прежнюю папку моделей в настройках, а пользователь может
позже выбрать другое место. `history.json` содержит полный текст фраз и не шифруется. Для удаления
всех сохранённых фраз откройте окно истории, нажмите «Очистить историю» и подтвердите действие.
По умолчанию временные WAV удаляются после обработки.

## Обновление прошивки

Готовое приложение содержит совместимый пакет прошивки. Подключите M5StickC Plus2 по USB, нажмите
«Прошивка…» и выберите найденный CH9102-порт.

- Обычное обновление записывает приложение по адресу `0x10000` и сохраняет Wi-Fi и настройки.
- Заводская установка стирает flash-память, записывает загрузчик и все разделы и удаляет настройки.
- Перед записью проверяются плата, версия и SHA-256 файлов; после перезагрузки приложение повторно
  читает версию устройства.

Автоматическая загрузка новой прошивки из GitHub Releases пока не реализована: новая версия
приложения или пакет прошивки скачиваются пользователем вручную.

## Запуск и сборка из исходников

Установите Python 3.11, 3.12 или 3.14 и PlatformIO Core. В PowerShell из корня репозитория выполните:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\setup_windows.ps1
.\scripts\start_tray.ps1
```

Запуск небольших автоматических тестов (они используют временные каталоги и не затрагивают реальные
настройки, модели, историю или подключённое устройство):

```powershell
.\.venv\Scripts\python.exe -m pytest -q
```

Сборка прошивки, Windows-приложения и чистого релизного архива:

```powershell
.\scripts\build_firmware.ps1
.\scripts\build_server_exe.ps1
.\scripts\package_release.ps1
```

Прошивка напрямую из исходников:

```powershell
.\scripts\flash_firmware.ps1
.\scripts\flash_firmware.ps1 -Port COM11
.\scripts\flash_firmware.ps1 -Factory
```

Номер COM-порта в репозитории не фиксируется; Windows может назначить другой номер.

## Устранение неисправностей

### Устройство не находится по USB

- Используйте кабель с передачей данных.
- Установите драйвер WCH CH9102.
- Закройте PlatformIO Serial Monitor и другие программы, занявшие COM-порт.
- Обновите список портов в окне прошивки.

### Устройство не подключается к Wi-Fi

Поддерживается только 2,4 ГГц. Проверьте SSID, пароль и уровень сигнала. После нескольких неудачных
запусков режим настройки откроется снова; его также можно вызвать удержанием B при включении.

### Wi-Fi подключён, но приложение не видит устройство

Проверьте, что компьютер и M5 находятся в одной частной сети, что в устройстве указан правильный
IPv4-адрес компьютера и что порт 8765 разрешён брандмауэром только для частной сети. Гостевые сети с
изоляцией клиентов не подходят.

### Не работает распознавание или вставка

Проверьте выбранную модель, активное текстовое поле и
`%LOCALAPPDATA%\VoxCortex\logs\server.log`. Параметр
`diagnostic: true` отключает клавиатурную вставку и оставляет подробную диагностику. Никогда не
публикуйте журнал, `history.json`, `config.yaml` или пароль Wi-Fi без предварительной очистки.
